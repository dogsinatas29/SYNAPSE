# 🚀 Release Notes - v0.2.11 "Multi-Language Intelligence"

## 🇺🇸 English

> **"Experience the depth of your architecture across Python, C/C++, and Rust."**

Version 0.2.11 is a milestone update that transforms SYNAPSE into a truly **Multi-Language Architecture Engine**. We've unified the intelligence level of our scanners to ensure that whether you are working on a Python web app, a high-performance C++ engine, or a safety-first Rust project, your architecture is visualized with precision.

### ✨ What's New?

#### 🧠 Unified Multi-Language Intelligence
We've significantly upgraded our core scanning engine to handle complex dependency resolutions across different ecosystems:
- **Python**: Deep import resolution for internal modules and automatic filtering of virtual environments.
- **C/C++**: Intelligent distinction between local headers (`"..."`) and system headers (`<...>`), ensuring only relevant project dependencies appear on your canvas.
- **Rust**: Enhanced `use` statement analysis, correctly identifying internal `crate`, `self`, and `super` references to map your crate structure accurately.

#### ➡️ Advanced Flow Analysis (Flow View)
The **Flow View** is no longer just for JavaScript and Python. It now supports:
- **C/C++ logic**: Detection of `if`, `while`, `for`, `switch`, and nested function calls.
- **Rust Idioms**: Full support for Rust's unique control structures, including `match` expressions and error handling patterns (`Result`/`Option`).

#### 🛡️ Architecture Integrity
- **External Node Mapping**: All supported languages now consistently identify and map external libraries, keeping your internal logic clean.
- **Accuracy Fixes**: Resolved issues where internal components were sometimes misidentified as external "ghost" nodes.

### 🛠️ Technical Improvements
- **Optimized Packaging**: Reduced VSIX overhead and improved extension activation time.
- **Rule Engine Sync**: Rules in `RULES.md` are now even more strictly applied across all scanners.

## 🇰🇷 Korean

> **"Python, C/C++, Rust를 아우르는 심층 아키텍처 분석을 경험하세요."**

버전 0.2.11은 SYNAPSE를 진정한 **"다중 언어 지능형 아키텍처 엔진"**으로 변모시킨 마일스톤 업데이트입니다. 모든 언어의 스캐너 지능을 상향 평준화하여, Python 웹 앱에서부터 고성능 C++ 엔진, 안정성 중심의 Rust 프로젝트에 이르기까지 당신의 설계를 정밀하게 시각화합니다.

### ✨ 주요 변경 사항

#### 🧠 통합 다중 언어 지능 (Multi-Language Intelligence)
핵심 스캐닝 엔진을 업그레이드하여 다양한 에코시스템의 복잡한 의존성 해석 기능을 강화했습니다:
- **Python**: 내부 모듈에 대한 심층 임포트 해석 및 가상 환경 파일의 지능적 필터링.
- **C/C++**: 로컬 헤더(`"..."`)와 시스템 헤더(`<...>`)를 지능적으로 구분하여, 프로젝트와 연관된 핵심 의존성만을 캔버스에 표시합니다.
- **Rust**: `use` 문 분석을 강화하여 `crate`, `self`, `super`와 같은 내부 참조를 정확히 식별하고 크레이트 구조를 완벽하게 매핑합니다.

#### ➡️ 확장된 플로우 분석 (Flow View)
이제 **Flow View**를 JavaScript와 Python뿐만 아니라 다음 언어에서도 사용할 수 있습니다:
- **C/C++ 로직**: `if`, `while`, `for`, `switch` 및 중첩된 함수 호출 감지.
- **Rust 패턴**: `match` 표현식과 `Result`/`Option` 핸들링 등 Rust 고유의 제어 구조를 완벽하게 지원합니다.

#### 🛡️ 아키텍처 무결성 보장
- **일관된 외부 라이브러리 매핑**: 지원되는 모든 언어에서 외부 라이브러리를 일관되게 매핑하여 내부 로직을 더욱 선명하게 보여줍니다.
- **노드 식별 정확도**: 내부 컴포넌트가 외부 "유령(Ghost)" 노드로 오인되던 문제를 해결했습니다.

### 🛠️ 기술 개선 사항
- **패키징 최적화**: VSIX 오버헤드를 줄이고 익스텐션 활성화 속도를 개선했습니다.
- **규칙 엔진 동기화**: `RULES.md`에 정의된 규칙들이 모든 스캐너에 더욱 엄격하게 적용됩니다.
